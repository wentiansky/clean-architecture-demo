# 奖励排序设计

## 背景

奖励页当前支持按状态筛选，但列表顺序保持接口原始顺序，用户难以快速看到即将过期的奖励。

## 目标

- 奖励列表支持“快过期优先”排序
- 没有过期时间的奖励排在最前面
- 仍保留现有筛选能力，排序在每个筛选结果内生效
- 同一排序权重下保持原始顺序，避免列表抖动

## 方案

- 在 `RewardService` 中新增统一排序方法
- `RewardPage` 的 `visibleRewards` 改为先筛选、再排序
- 排序规则：
  - `expireTime` 为空的奖励排最前
  - 其余奖励按 `expireTime` 升序排列，越快过期越靠前
  - 若两个奖励的排序键相同，则保留输入顺序

## 测试

- 服务层测试排序规则与稳定性
- 页面测试默认列表和筛选结果的展示顺序
