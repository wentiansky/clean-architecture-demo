# AGENTS

## 协作约定

- `GP` 的含义是：在相关验证通过后，执行 `git commit` 和 `git push`

## 执行规则

- 当用户说 `GP` 时，先运行与本次改动相关的验证命令
- 只有在验证通过后，才执行 `git commit` 和 `git push`
- 如果本次改动是纯文档修改，且没有对应的自动化验证命令，可以跳过测试
- 纯文档修改在执行 `GP` 前，仍需先检查 `git status` 或 `git diff`，确认变更范围正确
- 如果验证失败，停止提交和推送，并先反馈失败原因
- 如果本次改动还没有暂存或提交信息不明确，需要先整理后再执行 `GP`
